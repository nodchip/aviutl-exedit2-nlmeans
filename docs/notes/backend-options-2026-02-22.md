# GPU バックエンド候補比較メモ（2026-02-22）

## 前提

- 現行実装は `DirectX 11.3+`（`ID3D11Device3` 必須）。
- AviUtl ExEdit2 プラグインとして Windows 専用配布を想定。

## 候補比較

| 候補 | 長所 | 短所 | 現時点の判断 |
|---|---|---|---|
| DirectX 11.3+ (現行) | 既存コードを継続利用できる。導入コストが最小。 | 最新 GPU 機能への拡張性は限定的。 | 継続採用（基準実装） |
| DirectX 12 | 非同期実行やメモリ制御の自由度が高い。 | 実装複雑度が高く、保守コスト増。 | 複数GPU協調を本実装化する段階で再評価 |
| Vulkan | 将来的な移植性（他OS）を確保しやすい。 | ExEdit2 の現ターゲット（Windows）ではメリットが小さい。 | 当面見送り |
| DirectML | ノイズ除去の学習系手法へ拡張しやすい。 | 現行の非学習 NLM には過剰。依存が増える。 | 研究枠として保留 |
| OpenCL | GPU ベンダ横断を狙える。 | Windows環境でのドライバ差分吸収コストが高い。 | 当面見送り |

## 推奨方針

1. 短期: `DirectX 11.3+` 継続で Fast/Temporal を GPU 側へ展開する。
2. 中期: 複数GPU協調が必須になった時点で `DirectX 12` の PoC を作る。
3. 長期: ExEdit2 以外への展開要件が出た時点で `Vulkan` を再評価する。
