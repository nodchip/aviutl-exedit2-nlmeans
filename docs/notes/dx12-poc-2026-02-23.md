# DirectX 12 PoC メモ（2026-02-23）

## 目的

DirectX 11.3 実装を維持したまま、DirectX 12 移行の技術的リスクを段階的に下げる。

## 今回追加した最小実装

- `nlmeans_filter/exedit2/Dx12PocBackend.h`
  - `d3d12.dll` と `D3D12CreateDevice` の存在判定
  - UI 要求時に有効化可能かの判定関数
- `nlmeans_filter/exedit2/Dx12PocProcessor.h`
  - DX12 PoC 有効時に単一フレームを処理する最小経路（copy path）
  - 3x3 近傍平均の最小コンピュート相当経路（compute path）
  - D3D12 デバイス初期化可否の最小検証経路
  - d3dcompiler_47.dll を使った HLSL コンパイル可否の最小検証経路
  - D3D12 command queue/allocator/list 生成可否の最小検証経路
- `nlmeans_filter/tests/Dx12PocBackendTests.cpp`
  - ランタイム欠落 / エントリ欠落 / 利用可能の判定テスト
- `nlmeans_filter/tests/Dx12PocProcessorTests.cpp`
  - copy / compute 各経路の有効条件と出力整合テスト
  - D3D12 初期化可否検証ヘルパーの失敗系テスト
  - HLSL コンパイル可否検証ヘルパーの失敗系テスト
  - D3D12 コマンド生成検証ヘルパーの失敗系テスト
- `nlmeans_filter/tests/Dx11Dx12QualityReport.cpp`
  - DX11 出力と DX12 PoC（copy / compute）の品質比較レポート
- `nlmeans_filter/tests/Dx12PocBenchmark.cpp`
  - DX12 PoC（copy / compute）の単一フレーム平均時間レポート

## 現時点の方針

- 本番経路は引き続き DirectX 11.3 を使用する。
- DX12 PoC は「ランタイム検出 -> 単一フレーム処理 -> 画質比較 -> ベンチ比較」の順で拡張する。

## 次ステップ

1. DX12 PoC の compute path を HLSL 実装へ差し替え、CPU エミュレーション実装を置換する。
2. 協調実行（複数 GPU）に対するスケジューリング比較を行い、DX11 継続か DX12 移行かを判断する。
3. DX11 継続 / DX12 移行の判断基準（品質差、速度比、保守コスト）を `docs/operations/release-checklist.md` に明文化する。
